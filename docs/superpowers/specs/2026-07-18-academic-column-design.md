# 学术栏目与学术 Post 发布规范设计

日期：2026-07-18

## 目标

在现有 Jekyll Chirpy 博客中新增一级导航“学术”，与首页、分类、标签和归档处于同一导航层级。学术内容继续使用标准 Post，因此会正常出现在首页文章流、分类、标签和归档中；“学术”栏目只聚合带有 `学术` 标签的文章。

本次将 7 篇音频研究文献分别发布为 7 篇独立报告。原有综合报告删除，不保留为工作资料，也不作为额外文章发布。

今后新增相关学术内容继续遵循本文规范。

## 方案选择

采用“标准 Post + `学术` 标签 + 自定义一级聚合页”。

没有选择独立 Jekyll Collection，因为 Collection 默认不进入首页文章流，需要额外同步逻辑。没有仅使用 Category 页面，因为用户明确要求以 `学术` 标签作为栏目筛选依据。

## 导航设计

新增 `_tabs/academic.md`：

- 标题：`学术`
- 路径：`/academic/`
- 图标：`fas fa-graduation-cap`
- 顺序：位于“归档”之后
- 布局：使用 Chirpy 页面布局，并复用主题的文章列表视觉风格
- 数据来源：`site.tags["学术"]`
- 排序：按文章发布日期倒序
- 空状态：没有学术文章时显示简短说明，不产生模板错误

点击该栏目只展示包含 `学术` 标签的标准 Post。

## 学术 Post 规范

所有学术文章保存在 `_posts`，继续享受首页、永久链接、目录、评论、分类、标签和归档功能。

统一 Front Matter：

```yaml
---
title: "文章标题"
date: 2026-07-18 12:00:00 +0800
categories: [学术, 音频研究]
tags: [学术, ADQA]
description: "文章摘要"
toc: true
comments: true
---
```

约束如下：

- 每篇学术文章必须包含 `学术` 标签，这是进入栏目列表的唯一必要条件。
- 本批 7 篇文章同时包含 `ADQA` 标签。
- 根据论文主题增加专属标签，例如 `DCASE2025`、`S5`、`FSD50K`、`声源分离`、`音频质量评估`。
- Category 用于较宽泛的内容组织，本批统一为 `[学术, 音频研究]`。
- 文件名和永久链接使用稳定、可读的英文 slug，避免以后修改中文标题导致链接变化。
- 多篇同日发布时使用不同时间，保证首页和学术栏目排序稳定。

## 单篇报告内容结构

每篇报告采用统一但不过度模板化的结构：

1. 文献信息
2. 研究背景与核心问题
3. 方法或数据构建流程
4. 实验设置与关键结果
5. 核心贡献
6. 局限与批判性分析
7. 读后感
8. 论文带来的新思路
9. 对 ADQA 研究的启发
10. 总结

其中“读后感”“新思路”和“对 ADQA 的启发”必须针对该篇论文单独撰写，不能只复制跨论文的统一结论。涉及数值时以 PDF 表格和正文为准。

## 数据流

1. Jekyll 读取 `_posts` 中的标准文章。
2. 首页按既有 Chirpy 逻辑展示全部文章，包括学术文章。
3. 分类、标签与归档继续由主题和 `jekyll-archives` 生成。
4. `/academic/` 页面从 `site.tags["学术"]` 获取文章集合。
5. 用户点击列表项进入标准 Post 页面。

该设计只有一个内容源，不复制文章正文，也不需要维护额外索引数据文件。

## 删除与迁移

删除 `output/literature/ADQA_相关文献阅读与研究启发.md` 及不再需要的空目录。综合报告中的内容迁移并扩写到 7 篇 Post，迁移完成后不保留综合版本。

## 兼容性与异常处理

- 标签不存在时，学术页渲染空状态。
- 文章缺少 `学术` 标签时不会出现在学术栏目，但仍可作为普通 Post 发布。
- 标题和摘要中的英文缩写保留原论文写法。
- `ADQA` 在文章中明确说明当前采用的含义；如果以后课题定义改变，只更新相关 Post，不改变栏目机制。
- 不修改 Chirpy 主题 Gem 内部文件，避免主题升级覆盖改动。
- 不触碰现有未跟踪文章和用户的其他未提交修改。

## 验证方案

实施后执行以下检查：

1. 运行站点完整构建命令，确认 Jekyll 无 Liquid、Front Matter 或链接错误。
2. 检查 `/academic/` 成功生成且仅包含 7 篇带 `学术` 标签的文章。
3. 检查首页文章流包含这 7 篇文章。
4. 检查 `/tags/学术/` 和 `/tags/adqa/` 可访问。
5. 检查每篇文章的标题、目录、分类、标签和永久链接。
6. 检查学士帽图标和导航顺序。
7. 抽查所有关键实验数值是否与原 PDF 一致。
8. 确认综合报告已删除，且不存在第八篇重复文章。

## 完成标准

- 一级导航出现“学术”及学士帽图标。
- 学术栏目只显示带 `学术` 标签的文章。
- 7 篇报告都是标准 Post，并出现在首页文章流。
- 每篇报告可以独立阅读，包含读后感、新思路和 ADQA 启发。
- 综合报告彻底删除。
- 站点构建与链接检查通过。
